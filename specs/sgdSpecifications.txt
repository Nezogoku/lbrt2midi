SGD HEADER, 0x10 Bytes
    SGXD 0x00
    Offset to bank name 0x04
    Offset to sample data 0x08
    Length of sample data + 0x80000000 0x0C
    
    Consists of the following chunks:
        BUSS, audio mixing settings (?)
        RGND, soundbank definitions
        SEQD, sequence data
        WAVE, sample definitions
        WSUR, surround sound definitions (samples with 3+ channels)
        WMKR, ???
        CONF, configuration settings
        NAME, name definitions


BUSS (Optional) HEADER, 0x10 Bytes
    BUSS 0x00
    Length of BUSS chunk 0x04
    Null (Start of BUSS chunk) 0x08
    Number of buss pre-definitions (?) 0x0C

BUSS (Optional) PRE-DEFINITIONS, Variable Bytes
    ~Basically consists of values~
    ~And offsets to buss settings~

BUSS (Optional) SETTINGS, Variable Bytes
    ~Basically consists of strings of~
    ~Variables of some sort (buss_1/Tverb/UMix_ambi...)~
    ~And audio processing settings (wet/dry/mix...)~


RGND (Optional) HEADER, 0x10 Bytes
    RGND 0x00
    Length of RGND chunk 0x04
    Null (Start of RGND chunk) 0x08
    Number of program pre-definitions 0x0C
    
RGND (Optional) PROGRAM PRE-DEFINITIONS, 0x08 Bytes
    ??? 0x00
    Offset to program definition 0x04

RGND (Optional) PROGRAM DEFINITIONS, 0x38 Bytes
    Priority (?) 0x00
        If true, then range should probably be 0 to 127
    Group number (?) 0x04
        If true, then range should probably be 0 to 127
    Noise clock (?) 0x08
        Usually 56
        If true, then range should probably be 0 to 60
    Might be an indicator 0x0C
        0x00 is instrument (?)
        >0x00 is SFX (?)
    Unknown 0x10
    Unknown 0x14
    Lowest range of sample (note_low) 0x18
    Highest range of sample (note_high) 0x19
    Reserved 0x1A
    Root key of sample (centerNote) 0x1C
        0 to 127
        If greater than max, remaining used as semitune (centerDetune)
    Root key fine tuning, signed (detune) 0x1D
        -99 to 99
    Maximum Volume (?) (volume) 0x1E
        0 to 4096
    ADSR1 (?) 0x20
        Mostly seeing -127 to 127
    ??? 0x21
        Mostly seeing 0 to 16
    ADSR2 (?) 0x22
        Mostly seeing -100 to 100
    Pan Control, signed (panpot) 0x23
        Mostly seeing -4 to 4
        If true, then range should probably be -127(?) to 127
    Minimum Volume (?) (volume) 0x24
        0 to 4096
    Reserved (?) 0x26
    Portamento travel time (?) (portamentTime) 0x28
        0 to 65535
    Modifier 0 (?) 0x2C
    Modifier 1 (?) 0x2D
    Modifier 2 (?) 0x2E
    Modifier 3 (?) 0x2F
    ??? 0x30
    Corresponding sample ID 0x34
        0xFFFFFFFF is noise

SEQD (Optional) HEADER, 0x0C Bytes
    SEQD 0x00
    Length of SEQD chunk 0x04
    Null (Start of SEQD chunk) 0x08

SEQD (Optional) OFFSETS, Variable Bytes
    Number of offsets 0x00
    Offset 0x04[, 0x08, 0x0C,...]
        Null (Likely means go to next offset)
        Offset to Null (Sometimes used to separate sequences)
        Offset to offset

SEQD (Optional) SEQUENCES, Variable Bytes
    Offset to sequence name 0x00
    Sequence type 0x04
        0x00 is raw MIDI data
        0x01 is SFX controllers... somehow
    Sequence data 0x05
            Division is in FPS, 1SSSSSSS FFFFFFFF
            1 is division type FPS
            SSSSSSS is FPS
                24 FPS
                25 FPS
                29 FPS (Meaning 29.97)
                30 FPS
            FFFFFFFF is Ticks per Frame
        ~First byte is likely the MIDI format~
        Delta time 0xXX[XXXXXX]
            Variable length, ends with value < 0x80
            Sometimes has deltatime 0x00, though not always
        Status byte 0xXX
            0x8X noteNum velocity (Note off)
            0x9X noteNum velocity (Note on)
            0xAX noteNum pressure (Polyphonic aftertouch)
            0xBX value0 value1 (Control change)
                value0, 0x76 is custom defined
                value0, 0x77 is custom defined
                value0, 0x63 is toggle loop
                    value1, 0x14 is loop start
                    value1, 0x1E is loop end
            0xCX programNum (Program change)
            0xDX pressure (Channel aftertouch)
            0xEX 0xXX (Pitch bend)
            0xF1 0xXX (Time code)
            0xF2 0xXXXX (Song position)
            0xF3 songNum (Song select)
            0xF6 (Tune request)
            0xF8 (Timing clock)
            0xF9 (Measure end)
            0xFA (Start)
            0xFB (Continue)
            0xFC (Stop)
            0xFE (Active sensing)
            0xFF (Reset)
                0x00 0x00 (Sequence number)
                0x07 0xXX markerText (Marker)
                    markerText, text of variable length signifying synchronisation point at specific time
                0x2F 0x00 (End of track)
                0x51 0x03 0xXXXXXX (Change tempo)
                    0xXXXXXX, ms per quarter note; 0x000001 to 0xFFFFFF
                0x54 0x05 hours minutes seconds frames subframes (SMPTE offset)
                    hours, 0SSHHHHH
                        0 is reserved
                        SS is FPS
                            00 is 24 FPS
                            01 is 25 FPS
                            10 is 29 FPS
                            11 is 30 FPS
                        HHHHH is hours
                            0 to 23
                    minutes
                        0 to 59
                    seconds
                        0 to 59
                    frames
                        0 to (FPS - 1)
                    subframes
                        0 to 99
                0x58 0x04 numByte denByte midClocks 32perQuart (Time signature)
                    numByte, numerator of time signature
                    denByte, denominator of time signature ^ (-2)
                    clocksPerClick, MIDI clocks per metronome click
                    32perQuart, 32nd notes per 24 MIDI clocks
                0x59 0x02 keySig keyScale (Key signature)
                    keySig, Key signature; -7 to +7
                    keyScale, Key scale; 0 is major, 1 is minor


WAVE (Optional) HEADER, 0x10 Bytes
    WAVE 0x00
    Length of WAVE chunk 0x04
    Null (Start of WAVE chunk) 0x08
    Number of sample definitions 0x0C

WAVE (Optional) DEFINITIONS, 0x38 Bytes
    Voice (?) 0x00
        0 (?) to 8
        Just spit-ballin' here...
    Offset to sample name 0x04
        Null likely indicates SFX or note samples
            Follow order of name ID's in NAME section... somehow
    Codec 0x08
        0x01, PCM16 Big Endian (headerless)
        0x02, Ogg Vorbis
        0x03, Sony ADPCM (headerless)
        0x04, Sony Atrac3Plus
        0x05, Sony Short ADPCM (headerless)
        0x06, Sony Atrac3 (headerless)
    Number of channels 0x09
    Null 0x0A
    Sample rate 0x0C
    Info type of following value 0x10
        0x00, Null
        0x30 and 0x40, size of sample data
        0x60, ???
        0x80 and 0x90, block size of sample data
    Corresponding value to info type 0x14
    Maximum volume (?) 0x18
    Minimum volume (?) 0x1A
    Null 0x1C
    Length of sample 0x20
    Start of loop, signed 0x24
        -1, starts at beginning of sample, no loop
    End of loop, signed 0x28
        -1, ends at end of sample, no loop
    Length of sample stream or interleave 0x2C
    Offset to sample stream from start of sample data 0x30
    Length of sample stream (zero-padded) or interleave 0x34
